# ai-resume-assistant
🚀 基于大模型的 Tampermonkey 脚本，一键全自动智能填写各类招聘网站的网申简历表单AI-powered Tampermonkey script for one-click fully automatic intelligent filling of various online application resume forms on recruitment websites
# 🤖 AI 智能简历助手 (Tampermonkey 脚本)

![Version](https://img.shields.io/badge/version-2.4.0-blue.svg)
![License](https://img.shields.io/badge/license-MIT-green.svg)

基于大语言模型（LLM）的网页表单自动填写工具。只需粘贴一次你的纯文本简历，即可在各种招聘网站、企业官网的网申系统中，实现**一键全自动智能填表**。

## ✨ 核心特性

- **🤖 语义级精准识别**：突破传统固定规则提取，利用 AI 理解网页表单上下文（如“第一段经历”与“第二段经历”的区分）。
- **🖱️ 底层穿透注入**：模拟真实用户的点击、聚焦、输入、失焦等事件，绕过大部分前端框架（Vue/React）的输入限制。
- **🫧 极简悬浮 UI (V2.4 新特性)**：默认折叠为悬浮小圆圈，不遮挡网页视野；支持自由拖拽；支持 UI 状态及输入内容的本地记忆。
- **🔌 兼容主流模型**：采用标准 OpenAI API 格式，可自由配置接入 NVIDIA NIM、OpenAI、DeepSeek、阿里云百炼等任何兼容接口。

## ⚙️ 初始配置 (首次使用必看)

脚本默认使用 [NVIDIA NIM API](https://build.nvidia.com/) (Meta Llama 3.1 8B 模型) 进行推理。**你需要配置自己的 API Key 才能使用。**

1. 随便打开一个网页，页面右上角会出现一个 `🤖` 悬浮球，点击展开面板。
2. 点击面板标题栏右侧的 **⚙️ (设置图标)**。
3. 填入你的 API 配置：
   - **API URL**: 默认为 `https://integrate.api.nvidia.com/v1/chat/completions`（如果你用 OpenAI 或其他中转 API，请修改此处）。
   - **Model**: 默认为 `meta/llama-3.1-8b-instruct`（对应你使用的模型名称）。
   - **API Key**: 填入你申请到的对应平台的 API 令牌（如 `sk-xxxxxx`）。
4. 点击 **“保存配置”**，设置会自动隐藏。

## 📖 使用手册

### 第一步：准备简历
打开脚本面板，在多行文本框中粘贴你的**纯文本简历**（包含基本信息、教育经历、工作经历、项目经验等，越详细越好）。脚本会自动保存你输入的内容，下次打开无需重复粘贴。

### 第二步：一键填写
1. 进入任何你需要填写的网申简历页面（如企业招聘官网的资料填写页）。
2. 点击面板上的 **“⚡ 一键智能填写”** 按钮。
3. 按钮会提示 `AI 正在扫描表单特征...`，请耐心等待几秒钟。
4. 脚本会自动分析网页上的所有输入框、下拉菜单和文本域，并调用 AI 将你的简历信息逐一填入网页中。
5. 填写完成后，按钮会提示 `成功填充 X 项`。

### 💡 交互小技巧
- **拖拽**：鼠标按住顶部蓝色标题栏或悬浮小圆圈，可以在屏幕上任意拖拽面板位置。
- **折叠**：点击右上角的 **➖ (最小化图标)**，面板会收缩成一个不占空间的小圆圈。
- **人工核对**：AI 填写完成后，被修改过的输入框会亮起**浅绿色背景**，方便你快速核查填写是否准确。

## 📝 进阶技巧：如何制作“完美”的简历数据底本？

国内各大企业的网申系统（如北森、大汉三通等）往往要求填写非常细致的字段（如身份证号、紧急联系人、身高体重、户口所在地等），这些信息通常不会写在我们的 PDF 简历中。

为了让 AI 助手能 100% 填满网申表单，**强烈建议你先使用大模型（如 Kimi、ChatGPT、豆包等），将你的现有简历转化为“大而全的网申标准格式”。**

### 操作步骤

**第一步**：复制下方的【Prompt 提示词】和【通用简历信息模板】。
**第二步**：将它们和你的**个人真实简历**一起发给你常用的大语言模型（如 ChatGPT / Claude / Kimi）。
**第三步**：让大模型帮你整理并补全一份完整的文本。
**第四步**：将大模型生成的这份完整文本，粘贴到本脚本的输入框中作为长期的“数据底本”。
